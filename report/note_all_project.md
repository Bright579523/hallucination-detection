# 🛡️ Defense Questions: Engineering Rationale
(คำถามคลาสสิกสไตล์กรรมการสอบวิทยานิพนธ์ และแนวทางการตอบ)

นี่คือเหตุผลแบบเจาะลึกที่คุณสามารถนำไปใช้ตอบกรรมการหรือเขียนลงใน Report ได้อย่างมั่นใจครับ:

### 1. ทำไมต้องใช้ชุดข้อมูล COCO (Common Objects in Context)?
ไม่ได้เป็นเพราะมันแค่หาโหลดง่ายครับ แต่มีเหตุผลเชิงวิชาการที่แข็งแกร่งรองรับ:
* **มาตรฐานระดับโลก (Gold Standard):** ในวงการ Computer Vision COCO คือชุดข้อมูลมาตรฐานที่งานวิจัยระดับ Top-tier เกือบทุกชิ้นใช้ประเมินผล การที่เราใช้ COCO ทำให้ผลลัพธ์ของโปรเจกต์คุณมีความน่าเชื่อถือและสามารถนำไปเปรียบเทียบกับ Paper อื่นๆ ได้
* **ฟีเจอร์ "Supercategory" (อาวุธลับของโปรเจกต์นี้):** นี่คือเหตุผลสำคัญที่สุด COCO มีการจัดลำดับชั้นข้อมูล (Hierarchy) ไว้แล้ว เช่น หมวดหมู่สัตว์ (Animal) จะครอบคลุมทั้ง สุนัข แมว ม้าลาย ฯลฯ ทำให้คุณสามารถเขียนโค้ดสุ่ม **"Hard Negatives"** (เช่น เอารูปสุนัข มาหลอกโมเดลด้วยคำว่า หมาป่า) ได้แบบอัตโนมัติ โดยไม่ต้องไปนั่งหาข้อมูลหรือสร้าง Labels ใหม่เองเลย 

### 2. ทำไมต้องใช้โมเดลสำเร็จรูป OWL-ViT และ CLIP?
ในโปรเจกต์นี้เราต้องการสร้าง "ระบบตรวจสอบ (Verifier)" ไม่ใช่การสร้างโมเดลตรวจจับวัตถุแข่งกับ Google หรือ OpenAI การใช้ Pre-trained models จึงเป็นตัวเลือกที่ฉลาดที่สุดด้วยเหตุผลดังนี้:

* **OWL-ViT (The Target):** โมเดลเก่าๆ อย่าง YOLO หรือ Faster R-CNN จะตรวจจับได้แค่คลาสที่ถูกเทรนมาตายตัว (เช่น 80 คลาส) แต่ OWL-ViT เป็นโมเดล **Open-Vocabulary** ที่รับคำสั่งเป็นข้อความ (Text Prompt) อะไรก็ได้ให้ไปหาในภาพ ซึ่งความยืดหยุ่นตรงนี้นี่แหละที่ทำให้เกิด **"จุดอ่อนเรื่องการอุปโลกน์ (Hallucination)"** เราจึงต้องเลือกโมเดลประเภทนี้มาเป็นเป้าหมายในการทดสอบ
* **CLIP (The Independent Judge):** เราต้องการเครื่องมือมาช่วยเช็กผลลัพธ์ของ OWL-ViT แบบเป็นกลาง CLIP ถูกเทรนมาด้วยภาพและข้อความหลายร้อยล้านคู่เพื่อทำความเข้าใจ "ความหมาย (Semantics)" มันจึงทำหน้าที่เหมือน **กรรมการอิสระ** ที่คอยให้คะแนน (Cosine Similarity) ว่าสิ่งที่ OWL-ViT วงกรอบมาให้ กับข้อความที่ผู้ใช้พิมพ์ลงไป มันมีความสอดคล้องกันจริงๆ หรือแค่หลอนไปเอง

### 3. ข้อดีของ Google Colab
การเลือกใช้ Colab ตอบโจทย์การทำงานกลุ่มและข้อจำกัดทางทรัพยากรได้สมบูรณ์แบบ:
* **Collaboration (การทำงานร่วมกัน):** ตัดปัญหาเรื่อง "โค้ดรันผ่านในเครื่องฉัน แต่พังในเครื่องเพื่อน" (Dependency Hell) เพราะทุกคนทำงานบน Environment เดียวกันบนเบราว์เซอร์ และดึงไฟล์ภาพจาก Google Drive ก้อนเดียวกัน
* **Free GPU Power:** การประมวลผล Vision Transformer (ViT) ทั้งฝั่งภาพและข้อความบนรูปภาพ 5,000 รูปนั้น กินพลังการประมวลผลมหาศาล หากรันด้วย CPU ของแล็ปท็อปปกติอาจใช้เวลาเป็นวันๆ แต่ T4 GPU ฟรีของ Colab สามารถย่นเวลาให้จบได้ในหลักนาทีหรือชั่วโมงต้นๆ เท่านั้น

> **สรุป:** การผสมผสานระหว่างชุดข้อมูลที่มี Supercategory, โครงสร้างโมเดลแบบ Open-vocabulary, และแพลตฟอร์ม Cloud Computing ทำให้โปรเจกต์นี้ดูเป็น **Production-ready pipeline** ที่วิศวกรข้อมูลในบริษัท Tech ชั้นนำใช้งานกันจริงๆ ครับ
